\subsection{Detailed Description}
Holds the stats and pointers to the memory operations performed. You can create multiple contexts in an application if desired, so you can separate tracking between network, gui, etc.

By default, there's a single, global context used, which is automatically passed in by the custom macros we work with. To support additionals, you could create another macro like\-: 
\begin{DoxyCode}
\textcolor{preprocessor}{#define MALLOC\_NET(size)        tracked\_alloc(g\_net\_memctx, ...}
\textcolor{preprocessor}{...}
\end{DoxyCode}
 and provide your context as another extern.

Will not exist if U\-S\-I\-N\-G\-\_\-\-M\-E\-M\-O\-R\-Y\-\_\-\-D\-E\-B\-U\-G\-G\-I\-N\-G is not defined.

Most interaction with this class should be done with the special macros\-:
\begin{DoxyItemize}
\item M\-A\-L\-L\-O\-C
\item R\-E\-A\-L\-L\-O\-C
\item F\-R\-E\-E These handle providing the line number, file, and function, and will define to the real malloc/realloc/free automatically if not debugging memory.
\end{DoxyItemize}

Naturally, anything allocated/freed by malloc/realloc/free outside of our macros will not be tracked.

You must initialize the context with \hyperlink{tracked__memory_8c_a0a67a6685295c0e8f0d891fe56353b4d}{mem\-\_\-context\-\_\-init()}, and cleanup with \hyperlink{tracked__memory_8c_a18ea1d9235496d3eeefa9e5ef683a7a1}{mem\-\_\-context\-\_\-destroy()}.
